Paragraphe incluant des balises <br >

Les balises <br > permettent de sauter des lignes
Il n'est pas nécessaire de les fermer.

Néanmoins, pour une validation stricte ou si l'on veut parser du XML,
on peut fermer directement un saut de ligne dans la balise ouvrante comme ceci :
<br />

les balises HTML ne sont pas case sensitive.

<p> Un paragraphe </p>
affichera la même chose que <P> Un paragraphe </P>

Même si les majuscules sont tolérées dans les tags en HTML5

W3Schools recommande l'utilisation du lowercase

Les attributs HTML

L'attribut href

Un lien se définit par le tag <a>.
l'attribut href permet de spécifier l'url
Un lien vers le site w3schools

L'attribut src

C'est le tag <img> qui permet de
définir une image.
C'est dans l'attribut src que l'on donne le chemin vers l'image.

Il est possible d'ajuster la largeur et la hauteur en pixels de l'image dans les attributs width et height

L'attribut alt permet aux non voyants d'avoir une description de l'image interpréter par les screan readers.
De plus si l'image n'est pas disponible, ce sera la valeur de l'attribut alt qui sera affichée.

L'attribut lang

Pour une meilleure accéssibilité et pour les moteurs de recherche, il est bon de déclarer le langage de la page dans la balise <html> via l'attribut lang.

L'attribut title

Permet au passage de la souris sur un élément d'afficher un texte en info bulle. C'est le cas de ce paragraphe.

Importance de mettre les valeurs des attributs entre cote

Même si cela peut fonctionner sans, cela peut générer des comportements non désirés. Comme dans ce même paragraphe qui n'affiche qu'en partie l'info bulle faute de quotes. Conclusion : il faut toujours mettre les valeurs entres quotes.

Simple ou double?

Des double pour la plupart des cas sauf s'il y a déjà dans notre contenu des simples ou doubles quotes. On se rabattra sur des simples quotes et vice versa dans ces cas précis. Exemple dans le title de ce paragraphe.
